CUDA

Finding new ways to utalize the CUDA chip. Whether by using CUDA in the CUBE or integrating it in Mathematica.

Project Members: C. Boren, A. Dakkak, W. Davis, and J. Shaber.

Advisors: George Francis and Johnathan Manton

Overview

  • Fluidz: A split of FluidsGL
  • FluidzAR: A port of Fluidz to SZG
  • MandelbrotDy: A split of Mandelbrot
  • Cuda Mathematica(CM): Brining the Fluidz and MandelbrotDy client to Matica
  • Cuda Abstraction to Mathematica(CAM): Using CUDA as a coprocessor for Matica

Introduction

  • The GPU's speed has been increasing steadily over the past decade, while the amount of processor cores and their speed have been constant over the past few years.
  • The GPU's architecture is SIMD
  • Low price, while offering high performace (the democratization of super computers)
  • Cuda is a lanuage for the NVIDIA GPU

New

Since the summer, the NVIDIA was able to
  • Support more graphics cards
  • Release a new version of their SDK and Toolkit
  • Support for double precision on their 200-series cards
The community, on the other hand, was able to
  • Develop Java and Python bindings
  • Extend the linear algebra library
  • And more

Cuda Mathematica

  • Using Mathematica, rather than a custom OpenGL client, as the rendering engine for the client.
  • Used Mathlink and Mathematica 6
  • Ported both the MandelbrotDy and Fluidz clients to CM

Demonstration

Cuda Abstraction to Mathematica

  • Using Cuda as a computational engine for specific Mathematica functions.
  • Able to perform FFT/IFFT (via CUFFT) and Matrix Multiplication (via sgemm in CUBLAS).
  • Matlab had something similar for FFT, but their implementation was trivial since both Matlab's FFT and CUFFT are versions of FFTW
  • Alpha: Still Buggy

Lessons Learned

  • It is no fun writing a client/server RPC everytime you want to split a program.
  • The Cuda toolkit, at least version 1.1, has many bugs.
  • Writing Cuda programs is straigh forward, but optimization is tricky.

Future

  • Use Apache/Facebook's Thrift to generate RPC. Demo.
  • Install the new Toolkit and SDK.
  • Migrate code to the new SDK.
